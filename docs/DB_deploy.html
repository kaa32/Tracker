Установка базового ПО и модулей.

Для работы понадобится следующее базовое ПО:

  1. 1.БД postgres версии не ниже 9.2 (текущие поддерживаемые версии 9.2.4+ и 9.3.1+) 

  2. 2.Модуль postgis (текущие поддерживаемые версии 2.0+ и 2.1+) 

  3. 3.php версии не ниже 5.4 

  4. 4.Модули php-pgsql (драйвер postgresql), php-mcrypt(вызов функций шифрования), php-posix (используется для многопоточности). Также php должно поддерживать работу в режиме командной строки (cli - command line interface) из каталогов дистрибутива utils и ../utils. 

  5. 5.Для развертывания понадобятся каталоги distributive/utils и distributive/sql со всем их содержимым, а также файл distributive/DbConfig.php. 

  6. 6.Опционально для тестирования понадобятся ПО phpunit и каталог distributive/tests 

 

Настройка кластера БД

  1. 1.В кластере postgres рекомендуется выставить кодировку символов UTF8. Делается это следующими командами от суперпользователя (в бд с именем postgres):
    UPDATE pg_database SET datistemplate=FALSE WHERE datname='template1';
    DROP DATABASE template1;
    CREATE DATABASE template1 WITH owner=postgres template=template0 encoding='UTF8';
    UPDATE pg_database SET datistemplate=TRUE WHERE datname='template1'; 

  2. 2.В кластере должна быть создана база данных с именем test (или выбранным вами) и кодировкой 'UTF8' (она же 'UNICODE'). Кодировка будет выставлена в UTF8 автоматически, если пункт 1 выполнен успешно.  

Развертывание схемы

  1. 1.Настройка конфигуратора. Необходимо отредактировать файл DbConfig.php, установив параметры соединения с нужной базой данных. 

  2. 2.Запустить терминал, в терминале перейти в каталог utils и запустить команду 'php rebuild_schema.php' 

  3. 3.После развертывания схемы будет выведен список тестовых таблиц с тестовыми данными (по 5 записей на таблицу, последняя таблица - ALERT). Необходимо проверить отстутствие ошибок в выдаваемом результате. 

  4. 4.В процессе развертывания в БД будут созданы роли (пользователи БД) coder1, coder2, webconnect, wwwadmin с паролями привязанными к текущим настройкам системы. Рекомендуется проверить соответствие паролей режиму работы веб-фронтенда или сменить их на соответствующие. 

Очистка тестовых данных.

  1. 1.После развертывания можно очистить тестовые данные SQL-командой
    DELETE * FROM msc.users; от обычного пользователя.
    Будут очищены все данные, кроме тарифов и биллинговых записей.
