cron1()

CREATE OR REPLACE FUNCTION msc.cron1() RETURNS INTEGER

Stored procedure to be called every 1 hour (or at similar period).  Procedure updates tariffs and geozones from last call of that cron1.  It does not use CRON1_INTERVAL parameter for safety reason.

Returns

INTEGERnumber of processed rows

Example

select msc.cron1();

cron24()

CREATE OR REPLACE FUNCTION msc.cron24() RETURNS VOID

Very important stored procedure to be called every 24 hour (or at similar period).  Creates partitioned tables for alerts and geodata.  Also drops old geodata and alert partitioned tables.  Do not call from superuser!  (Admin will can not remove superuser’s objects )

Returns

VOIDnumber of processed rows

See also

msc.constants

Example

select msc.cron24();

cron72()

CREATE OR REPLACE FUNCTION msc.cron72() RETURNS INTEGER

Stored procedure to be called every 72 hour (or at similar period).  Procedure deletes confirmation that is older than CRON72_INTERVAL.  Procedure calculates user_last_contact time for all free users and set last_contact_time to premium and banned users to current time.  Procedure kills all inactive users

Returns

INTEGERnumber of processed rows

msc.constants

Example

select msc.cron72();
