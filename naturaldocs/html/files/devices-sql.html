insert_new_device(int, text, text, text, bool, bool)

CREATE OR REPLACE FUNCTION msc.insert_new_device(
   uid int,
   dname text,
   duuid text DEFAULT NULL,
   dver text DEFAULT NULL,
   is_silent BOOLEAN DEFAULT FALSE,
   is_deaf BOOLEAN DEFAULT FALSE
) RETURNS INTEGER

Inserts new device into table msc.devices.  Generates alert DEVICE CREATED on successful return.  Alert generation suppressed if device_name is ‘BROWSER_DEVICE01’.  If user has no master-device or master-device is a browser_device then set created device as user-master-device.

Parameters

uid (int)identifier of user that owns device
dname (text)name of device
duuid (text)16-byte UUID of device DEFAULT NULL
dver (text)device version (up to 32 char) DEFAULT NULL
is_silent (bool)is_silent status for newly creating device, DEFAULT FALSE
is_deaf (bool)is_deaf status for newly creating device, DEFAULT FALSE

Returns

INTEGERidentifier of newly created device (or 0 for fail case)

Examples

select msc.insert_new_device(1, ‘device-name’);

link_device_to_user(int, int)

CREATE OR REPLACE FUNCTION msc.link_device_to_user(did int,
uid int) RETURNS INTEGER

Links device to user by setting device_user_id to user_id.  Generates alert DEVICE LINKED after successful linking.  Does not generate alert if device is always owned by user.  If device was master device for previous owner then set user_master_device of previous owner to BROWSER_DEVICE01.  Suppressed alert generation for BROWSER_DEVICE01.

Parameters

did (int)device identifier
uid (int)user identifier

Returns

INTEGER1 if success, 0 if failure

Examples

select msc.link_device_to_user(1, 1);
