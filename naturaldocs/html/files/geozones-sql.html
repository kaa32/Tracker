insert_new_geozone(int, int, text, text, float, float, float, text)

CREATE OR REPLACE FUNCTION msc.insert_new_geozone(
   uid int,
   cid int,
   zname text,
   zdesc text,
   zcenter_lat float,
   zcenter_lon float,
   zradius float,
   ktime text DEFAULT ((CURRENT_TIMESTAMP AT TIME ZONE 'UTC') + INTERVAL '1 day')
) RETURNS INTEGER

Inserts new geozone into table msc.geozones.  Function denies geozone creation if user tariff does not allow that.  Before inserting it checks if user has no tariff limitations.  If limits are reached then function generates alert LIMITS REACHED and return 0.  Function generates fields ‘geozone_data’ (JSON) and ‘geozone_geography’ (GEOGRAPHY) from parameters.  After inserting it create a record in table msc.zconsumers for a geozone owner.  Also generates alert GEOZONE CREATED on successful return.

Parameters

uid (int)identifier of user.
cid (int)identifier of geozone.
zname (text)name of geozone, unique per user
zdesc (text)description of geozone.
zcenter_lat (float)latitude of geozone center location
zcenter_lon (float)longitude of geozone center location
zradius (float)geozone radius in meters
ktime (text)text representation of timestamp when geozone should be deleted. default CURRENT_TIMESTAMP + INTERVAL ‘1 days’.  Recommended format of time ‘YYYY-MM-DD HH:MI:SS’.  Time of day is optional.  If ktime is NULL then geozone will never be deleted automatically.

Returns

INTEGERidentifier of newly created geozone (or NULL for fail case)

See also

msc.geozones.  JSON-format of ‘geozone_data’

constants.sql

Examples

select msc.insert_new_geozone(1, 1, ‘geozone-name’, ‘geozone-description’, 12.345, 67.89, 500, ‘2013-12-31 23:59:59’);
