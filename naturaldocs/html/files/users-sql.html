insert_new_user(text, text, text, text, text)

CREATE OR REPLACE FUNCTION msc.insert_new_user(
   ulogin text,
   uname text,
   upass_hash text,
   urec_tel text,
   utariff_name text DEFAULT 'free'
) RETURNS INTEGER

Inserts new user into table msc.users.  Raises exception if ulogin is shorter than 6 symbols.  Raises exception if uname is shorter than 4 symbols.  Generates alert USER CREATED on successful return.  Automatically creates circle with name ‘uname + circle’ with user as master.  Automatically creates billing record for user with free tariff and activate free tariff.  Automatically creates device with name ‘BROWSER_DEVICE01’ for that user.

Parameters

ulogin (text)user login, UNIQUE
uname (text)user name, UNIQUE
upass_hash (text)user password hash
urec_tel (text)user recovery phone
utariff_name (text)name of start tariff, DEFAULT ‘free’

Returns

INTEGERidentifier of newly created user (or 0 for fail case)

Examples

select msc.insert_new_user(‘iv.nosp@m.an@gmai.nosp@m.l.com’, ‘ivan’, ‘password-hash’, ‘322223’);

Exceptions

’insert_new_user() error: user_login too short’.

’insert_new_user() error: user_name too short’.

select msc.insert_new_user(‘ma.nosp@m.ria@gmai.nosp@m.l.com’, ‘maria’, ‘password-hash’, ‘322223’, ‘test-unlimited’);

get_user_location_history(uid int,

start_time text, end_time text default NULL) Returns location history for user by user identifier, start time and end time.  If parameter ‘end_time’ is not sent or is set to NULL then current time is used for t_end.  Parameters start_time and end_time should be provided AT TIME ZONE ‘UTC’.  Format of start_time and end_time is a timestamp-compatible text (‘YYYY-MM-DD[ HH:MI:SS]’ is recommended).

Parameters

uid (int)user identifier.
start_time (text)start of period in timestamp-format.
end_time (text)end of period in timestamp-format.  Default to NULL.

Returns

setof row(json, text)set of pairs(geodata location, geodata_timestamp).

Examples

select * from msc.get_user_location_history(1, ‘2013-10-01’);

select * from msc.get_user_location_history(1, ‘2013-11-10 08:00’, ‘2013-11-10 11:00’);

set_user_billrec(int, int)

CREATE OR REPLACE FUNCTION msc.set_user_billrec(uid int,
bid int) RETURNS INTEGER

Sets user billing record to new value (activate new tariff for user).  Raises exception if user tries to use not his/her own billing record.  If current time inside time interval of billrecord then generates alert TARIFF ACTIVATED for that user and removes excessive geozones and consumers for new tariff.  If new tariff denies device silent and deaf settings then all user devices set to non-silent and non-deaf.

Parameters

uid (int)user identifier
bid (int)new billing record identifier

Returns

INTEGER1 if billrec was updated, 0 if fails

Examples

select msc.set_user_billrec(1, 1);

CREATE OR REPLACE FUNCTION msc.set_user_active(uid int,

Sets user ‘is_active’ status to new value.  Generates alert USER ACTIVATED or USER DEACTIVATED depending on ‘is_active’ value.  In TRUE case calls update_user_billrec(int).  In FALSE case calls set_geozone_disabled(int) for all user geozones.

Parameters

Returns

INTEGER1 if success, 0 if failure
